# MatchingResultPage 및 ClubRecommendationPage 구현 계획서

## 📋 개요
`project-documentation.html`의 화면 정의서 및 플로우에 따라 성격 테스트 결과 이후의 매칭 결과 페이지(`MatchingResultPage`)와 모임 추천 페이지(`ClubRecommendationPage`)를 구현합니다.

## 📱 화면 정의 (참조: project-documentation.html)

### 1. 매칭 결과 페이지 (MatchingResultPage)
- **화면 ID**: - (TestResultPage 다음)
- **화면명**: 매칭 결과
- **주요 UI 요소**:
  - **헤더**: 뒤로가기 버튼, 타이틀 "당신과 비슷한 사람들"
  - **비슷한 성격 유형 카드**: 가로 스크롤 (ListView.horizontal)
  - **모임 카테고리 통계**: 2x2 그리드 (GridView)
  - **일러스트**: 페이지 분위기에 맞는 일러스트
  - **네비게이션 인디케이터**: 1/2 표시
  - **CTA 버튼**: "나에게 맞는 모임 찾기" (Primary Color)
- **액션**: 버튼 클릭 시 `ClubRecommendationPage` (#5)로 이동

### 2. 모임 추천 TOP 3 페이지 (ClubRecommendationPage)
- **화면 ID**: #5
- **화면명**: 모임 추천 TOP 3
- **주요 UI 요소**:
  - **헤더**: 뒤로가기 버튼, 타이틀 "당신을 위한 모임 TOP 3"
  - **모임 카드 (3개)**:
    - 모임 이모지 + 이름
    - 매칭률 뱃지 (예: 92%, 88%, 81%) - **Success Color**
    - 참여인원, 평점
    - 설명 (Description)
    - 태그 (위치/빈도/규모)
  - **네비게이션 인디케이터**: 2/2 표시
  - **CTA 버튼**: "프로필 만들고 모임 참여하기"
- **액션**: 버튼 클릭 시 프로필 생성 페이지(#6)로 이동 (Sprint 2 범위)
